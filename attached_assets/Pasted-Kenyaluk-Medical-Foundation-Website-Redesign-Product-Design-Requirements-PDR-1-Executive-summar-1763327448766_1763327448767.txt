Kenyaluk Medical Foundation Website Redesign – Product Design Requirements (PDR)
1  Executive summary

Kenyaluk Medical Foundation’s existing website (kenyalukmedicalfoundation.org) provides basic information about the organization’s programs, contact details and a portal login. However, the site is dated and has numerous shortcomings: the navigation is cluttered, many calls‑to‑action are buried, it lacks a dedicated events section with payments, and it does not appear to implement modern security or accessibility standards. The foundation’s stakeholders—donors, volunteers, sponsors, beneficiaries and partners—need a streamlined digital experience that inspires trust, communicates impact and makes it easy to act.

This Product Design Requirements (PDR) document leverages the Website Revitalization Proposal prepared by More Life Consulting on 11 Nov 2025 to define a clear scope for rebuilding the site on Replit. It also integrates current best‑practice research for nonprofit websites to ensure the new design delivers measurable results. Key goals include migrating the site to HTTPS, simplifying navigation, creating clear calls‑to‑action, optimizing pages for speed and mobile users, introducing secure donation and event registration flows, and ensuring the entire experience is inclusive and accessible. By following this PDR, the project is expected to increase donations, volunteer engagement and event participation while improving the foundation’s credibility and search visibility.

2  Current site assessment
2.1 Navigation and content structure

The home page currently displays a long vertical list of sections—images, mission statements, program summaries and various calls‑to‑action. The navigation menu includes numerous links (Home, About Us, Donate, Programs, Events, Medical Mission Fair, Projects, Gallery, Volunteer, Sponsorship and Contact Us). This quantity of links makes it difficult for visitors to find the most important actions quickly. The site also uses a mixture of internal links like “Click Here” rather than descriptive buttons, which can harm accessibility and SEO.

2.2 Visual design

The current design is busy, with large carousels and overlapping images. Colors, typography and spacing are inconsistent; there is limited white space and the site is not optimized for different screen sizes. Calls‑to‑action (e.g., “Donate”) are not prominent at the top of the page and do not use engaging language. There is also no unified style guide or brand definition; different pages appear disjointed.

2.3 Performance, security and accessibility

The existing site is served via HTTP rather than HTTPS (no lock icon), so data transmitted through forms is not encrypted. There may be broken links and unoptimized images that slow down loading. Research on nonprofit web design stresses the importance of fast page loads because more than half of visitors abandon a mobile site that takes longer than three seconds
wildapricot.com
. The site does not display accessibility features such as adjustable font sizes, alt text descriptions or high‑contrast options.

2.4 Functionality

Donation process: The current Donate link opens an external third‑party form; the user experience is disconnected and unclear. It lacks preset giving amounts, recurring donation options or simplified form fields.

Events: There is an “Events” link but no dedicated, searchable events page with registration and payment integration.

Volunteer & Sponsorship forms: Sign‑up forms are not clearly visible, and there is no automation to acknowledge submissions.

Portal login: A “Portal Login” link appears without explanation of its purpose.

Impact storytelling: There are few stories or statistics demonstrating how donations and programs change lives. Modern nonprofits should highlight impact stories and metrics to build trust
wildapricot.com
.

Overall, the site needs a modern redesign that aligns with best practices: clear mission and value proposition, consistent branding, simple navigation, prominent calls‑to‑action, easy donation and registration forms, and full accessibility

.

7  Technical architecture
7.1 Platform and tools

Replit environment: Use Replit’s workspace as the development, staging and deployment environment. A Git repository will allow version control and collaboration.

Framework: Use a modern JavaScript framework such as Next.js (React) for server‑side rendering and static site generation or SvelteKit for fast performance. Alternatively, a lightweight static site generator like Hugo or 11ty can be chosen if dynamic features are limited.

Styling: Employ a utility‑first CSS framework such as Tailwind CSS for rapid development and consistent styling. Combine it with accessible UI components or design system libraries.

CMS options: For easy content management, integrate an API‑based CMS (e.g., Sanity, Contentful) or Replit’s built‑in storage. This allows non‑developers to update programs, events and blog posts.

Payments: Integrate Stripe for secure donations and event payments. Stripe offers PCI‑DSS compliant processing and supports Apple Pay, Google Pay and other methods. Use the Stripe Checkout or Stripe Payment Links API for quick setup.

Email automation: Use services like SendGrid or Mailchimp to send donation receipts, event confirmations and volunteer follow‑ups. Use webhooks to trigger emails automatically.

Analytics & SEO: Implement Google Analytics 4 or Matomo for tracking. Use SEO plugins or manual meta tag configuration. Generate an XML sitemap and submit it to search engines.

7.2 Security and compliance

HTTPS & SSL: Obtain and install an SSL certificate to serve the site via HTTPS. This encrypts all data and signals to visitors that their information is secure
morweb.org
.

PCI‑DSS compliance: Ensure that donation and event payment processing complies with PCI standards. Do not store sensitive card data on the site; use Stripe’s secure tokens
morweb.org
.

Data protection: Adhere to privacy regulations by providing a transparent privacy notice and cookie policy. Collect only necessary data and store it securely.

Updates and patching: Keep all dependencies up to date. Use automated dependabot or similar tools to monitor vulnerabilities.

9  Success metrics and ROI

The success of the redesign will be measured using key performance indicators (KPIs) aligned with the project goals:

Conversion rates: Track the percentage of visitors who donate, register for events or sign up to volunteer. Aim for a 15–30 % increase in donations and event registrations (proposal estimate).

Traffic and SEO: Monitor organic traffic growth (target 20 % increase) by measuring search impressions and clicks. Track the ranking of branded and program‑related keywords.

Engagement: Measure average session duration, bounce rate and pages per session. Use blog engagement (comments and shares) as a proxy for interest.

Operational efficiency: Reduce administrative time spent on event management and form processing by 5–10 hours per month through automation.

Accessibility compliance: Use tools like the WAVE Web Accessibility Evaluation Tool to achieve WCAG 2.1 AA scores and monitor error reductions.

Support feedback: Collect feedback from donors, volunteers and staff on ease of use, clarity and satisfaction.

11  Conclusion

This PDR outlines a comprehensive roadmap for redesigning the Kenyaluk Medical Foundation website on Replit. By following the phases and requirements defined herein, the project will deliver a modern, secure, mobile‑friendly and accessible website that communicates the foundation’s mission effectively and drives action. Incorporating best practices—clear CTAs, storytelling, responsive design, optimized donation flows, event registration, and robust security—will provide a measurable return on investment and lay the groundwork for sustained growth in donations, volunteer participation and community impact.
